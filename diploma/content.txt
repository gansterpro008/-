H1:ВВЕДЕНИЕ
P:Актуальность темы дипломной работы обусловлена возрастающей потребностью предприятий малого и среднего бизнеса в автоматизации складского учёта. Современные экономические условия требуют от организаций оперативного и точного учёта товарно-материальных ценностей. Складской учёт является одним из ключевых бизнес-процессов, от эффективности которого напрямую зависит бесперебойность производства и финансовая стабильность предприятия.

P:На сегодняшний день на рынке программного обеспечения представлено множество решений для автоматизации учёта. Однако существующие продукты часто имеют избыточный функционал, высокую стоимость лицензирования и сложность внедрения. Это делает их недоступными для малого бизнеса. Разработка собственной информационной системы, ориентированной на конкретные потребности предприятия, позволяет получить оптимальное соотношение функциональности и стоимости.

P:Целью дипломной работы является разработка информационной системы [Монетка] для автоматизации складского учёта на предприятиях малого и среднего бизнеса. Для достижения цели необходимо решить задачи: провести анализ предметной области и существующих решений; разработать техническое задание; спроектировать архитектуру и базу данных; реализовать программное обеспечение с использованием C#, .NET 8, PostgreSQL; провести тестирование.

P:Объектом исследования является автоматизация складского учёта. Предметом исследования - процесс разработки информационной системы. Практическая значимость работы заключается в возможности непосредственного использования созданной системы для автоматизации складского учёта. Система позволит сократить временные затраты, минимизировать количество ошибок и обеспечить оперативный контроль остатков.

P:Работа состоит из введения, пяти глав, заключения, списка литературы и приложений. В первой главе проводится анализ предметной области. Вторая глава посвящена проектированию. Третья глава описывает реализацию. Четвёртая глава содержит тестирование. Пятая глава оценивает экономическую эффективность.

H1:1. АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ
H2:1.1 Описание бизнес-процессов складского учёта
P:Складской учёт включает комплекс операций по приёмке, хранению и отпуску товарно-материальных ценностей. Основными операциями являются: приёмка товаров от поставщиков с проверкой количества и качества; учёт текущих остатков; отпуск товаров по заявкам; проведение инвентаризации; формирование отчётности. Каждая операция требует документального оформления и отражения в учётной системе.

P:В процессе анализа выделены три роли пользователей. Администратор управляет системой и пользователями. Кладовщик выполняет основные складские операции. Менеджер анализирует отчёты и принимает управленческие решения. Такое разграничение обеспечивает безопасность данных и чёткое распределение ответственности.

P:Диаграмма вариантов использования включает акторов Администратор, Кладовщик, Менеджер. Администратор управляет пользователями и настройками. Кладовщик ведёт товары, создаёт накладные, проводит инвентаризацию. Менеджер просматривает отчёты об остатках, движении товаров и оборотную ведомость.

H2:1.2 Обзор существующих решений
P:На рынке представлены системы 1С:Склад, МойСклад, WMS-решения. 1С:Склад является наиболее распространённым решением, но требует значительных затрат на лицензирование и настройку. МойСклад предлагает облачный сервис с ежемесячной подпиской, но зависит от качества интернет-соединения. WMS-системы ориентированы на крупные складские комплексы и имеют высокую стоимость внедрения.

P:Для малого и среднего бизнеса оптимальным является создание собственной информационной системы, учитывающей специфику конкретного предприятия. Такой подход позволяет получить необходимый функционал без избыточных затрат и с возможностью дальнейшей адаптации под изменяющиеся потребности.

H2:1.3 Требования к системе
P:Функциональные требования: учёт товаров с указанием наименования, артикула, категории, цены и количества; ведение приходных и расходных накладных; автоматический контроль остатков; проведение инвентаризации; формирование отчётов; разграничение прав доступа. Нефункциональные требования: многопользовательский режим; время отклика до 2 секунд; резервное копирование; экспорт в Excel.

H1:2. ПРОЕКТИРОВАНИЕ ИНФОРМАЦИОННОЙ СИСТЕМЫ
H2:2.1 Архитектура системы
P:Информационная система [Монетка] реализована по клиент-серверной архитектуре. Серверная часть разработана на платформе .NET 8 с использованием языка C#. База данных PostgreSQL. Доступ к данным через Entity Framework Core. Клиентское приложение на Windows Forms. Архитектура включает три уровня: представление, бизнес-логика, доступ к данным.

P:На уровне представления расположены формы Windows Forms, обеспечивающие взаимодействие с пользователем. Уровень бизнес-логики содержит сервисы, реализующие основные функции. Уровень доступа к данным включает репозитории и контекст базы данных. Такое разделение обеспечивает гибкость и удобство сопровождения.

P:Диаграмма компонентов содержит: PresentationLayer с формами LoginForm, ProductListForm, InvoiceForm, InventoryForm, ReportForm; BusinessLogicLayer с сервисами; DataAccessLayer с репозиториями; Database PostgreSQL.

H2:2.2 Проектирование базы данных
P:База данных спроектирована в третьей нормальной форме. Таблица Users содержит идентификатор, логин, хэш пароля, роль, ФИО. Таблица Products - идентификатор, наименование, артикул, категория, единица измерения, цена, количество. Таблица Categories - идентификатор и наименование. Таблица Invoices - идентификатор, номер, дата, тип, контрагент, пользователь, сумма, статус.

P:Таблица InvoiceItems содержит идентификатор, ссылку на накладную, ссылку на товар, количество, цену. Таблица Inventory - идентификатор, дата, статус, пользователь, товары с фактическим количеством. Связи между таблицами организованы через внешние ключи. Индексы на полях Name и Article ускоряют поисковые запросы.

H2:2.3 Проектирование интерфейса
P:Интерфейс на Windows Forms с единым стилем. Главное окно содержит меню и панель инструментов. Таблицы отображаются в DataGridView с сортировкой. Формы ввода содержат валидацию полей.

H1:3. РЕАЛИЗАЦИЯ ИНФОРМАЦИОННОЙ СИСТЕМЫ
H2:3.1 Стек технологий
P:Язык C# 12, платформа .NET 8, среда Visual Studio 2022, СУБД PostgreSQL 16, ORM Entity Framework Core 8, клиент Windows Forms, библиотека EPPlus для отчётов. Выбор обоснован производительностью, бесплатностью и развитой экосистемой.

H2:3.2 Реализация модулей
P:Модуль авторизации через LoginForm. Пользователь вводит логин и пароль. UserService проверяет данные в БД. Пароли хранятся в виде SHA256-хэша. После аутентификации определяется роль и открывается главное окно.

P:Модуль товаров через ProductListForm с DataGridView. CRUD-операции. Поиск по наименованию. Категории выбираются из ComboBox. Валидация полей на стороне клиента.

P:Модуль накладных через InvoiceForm. Шапка: номер, дата, контрагент. Табличная часть: товары, количество, цена. При проведении проверяется остаток и обновляются количества.

P:Модуль инвентаризации через InventoryForm. Ввод фактического количества. Расчёт отклонений. Формирование акта.

P:Модуль отчётов формирует отчёт об остатках, о движении товаров, оборотную ведомость. Экспорт в Excel через EPPlus.

H1:4. ТЕСТИРОВАНИЕ И ОБЕСПЕЧЕНИЕ КАЧЕСТВА
P:Проведено функциональное тестирование всех модулей. Проверены сценарии авторизации, CRUD, создания накладных, инвентаризации, формирования отчётов. Граничные случаи: расход при нулевом остатке, отрицательные значения, доступ без прав. Все сценарии отработаны корректно.

P:Unit-тесты написаны с использованием xUnit. Протестированы сервисы UserService, ProductService, InvoiceService. Покрытие кода тестами составляет 72%. Система готова к эксплуатации.

H1:5. ЭКОНОМИЧЕСКАЯ ЭФФЕКТИВНОСТЬ
